# 3.3 Пример страницы скрипта

> Ниже приведен минимально хороший пример страницы рабочего ScriptRunner-скрипта для интеграционного scheduled job.

## Summary

Скрипт синхронизирует объекты из внешней системы в Jira Assets. Запускается по расписанию каждые 30 минут. Основной результат — создание и обновление карточек объектов без ручного вмешательства.

## Назначение

Скрипт нужен для автоматической актуализации данных в Assets и снижения ручной работы команды поддержки.

## Тип скрипта и точка запуска

- Тип: `Scheduled Job`
- Запуск: каждые 30 минут

## Владелец и контекст

- Владелец: команда Jira
- Контекст: общий интеграционный скрипт
- Критичность: высокая

## Путь к коду

- `scripts/shared/integrations/assets/syncAssetsFromExternalSystem.groovy`

## Входные данные

- данные API внешней системы;
- настройки подключения;
- маппинг полей.

## Выход или эффект

- создаются новые объекты;
- обновляются существующие объекты;
- ошибки записываются в лог.

## Зависимости

- внешний API;
- объекты Assets;
- общая библиотека маппинга;
- логгер `com.inforce.logger`.

## Логирование

- `INFO` — старт и завершение синхронизации;
- `WARN` — частичные проблемы;
- `ERROR` — ошибки обработки.

## Проверка после изменения

1. Запустить скрипт на тестовом наборе данных.
2. Проверить создание новых объектов.
3. Проверить обновление существующих объектов.
4. Проверить, что ошибки пишутся в лог корректно.

## Риски и ограничения

- зависимость от доступности внешнего API;
- риск некорректного маппинга;
- высокая чувствительность к изменению структуры входных данных.
