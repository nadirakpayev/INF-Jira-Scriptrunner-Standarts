# 3.3 Пример страницы скрипта: Mail Handler

> Ниже приведен пример страницы для event-driven ScriptRunner-сценария на основе логики уровня `Mail Handler for B2B`.

## Summary

Скрипт обрабатывает входящее письмо, ищет ключ Jira-задачи в теме письма и добавляет письмо в найденную задачу. Запускается при получении письма. Основной результат — автоматическая привязка входящей переписки к нужной Jira-задаче без ручной обработки.

## Назначение

Скрипт нужен для автоматической обработки входящих писем и снижения ручной работы команды при разборе почтовых обращений.

## Тип скрипта и точка запуска

- Тип: `Mail Handler`
- Запуск: при поступлении входящего письма

## Владелец и контекст

- Владелец: команда Jira
- Контекст: операционный сценарий
- Критичность: средняя

## Путь к коду

- `scripts/operations/mail-handlers/b2b/handleIncomingB2BMail.groovy`

## Входные данные

- тема письма;
- тело письма;
- вложения;
- служебные заголовки письма.

## Выход или эффект

- письмо добавляется в комментарии нужной Jira-задачи;
- вложения прикрепляются к задаче;
- если ключ задачи не найден, сценарий пишет предупреждение в лог.

## Зависимости

- почтовый обработчик Jira / ScriptRunner;
- API Jira для работы с задачами и вложениями;
- правила поиска ключа задачи в теме письма;
- логгер `com.inforce.logger`.

## Логирование

- `INFO` — старт обработки письма и успешное завершение;
- `WARN` — письмо не содержит ключ Jira-задачи или не прошло фильтрацию;
- `ERROR` — письмо не удалось привязать к задаче из-за ошибки выполнения.

Примеры:

- `Mail handler started: subject=[B2B] INC-4421 Request update`
- `Mail handler skipped message: Jira key not found in subject`
- `Mail handler finished: issueKey=INC-4421, attachments=2`

## Проверка после изменения

1. Отправить тестовое письмо с корректным ключом Jira-задачи в теме.
2. Проверить, что письмо добавилось в нужную задачу.
3. Проверить, что вложения прикрепились корректно.
4. Отправить письмо без ключа задачи и убедиться, что сценарий не падает, а пишет понятный `WARN`.

## Риски и ограничения

- зависимость от формата темы письма;
- риск неверного разбора ключа задачи;
- риск повторной обработки одного и того же письма;
- возможные ограничения по вложениям и размеру письма.
